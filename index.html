JavaScript: Control Flow - Loops

September 10, 2023

In the realm of application logic, loops reign supreme, determining what happens, how it unfolds, and for how long. Dive into the symphony of loops, where various methods lead to the same destination in handling iterative tasks. Among the loops, one particularly handy tool stands out...

for Loops

When it comes to arrays, the for loop takes the spotlight, enabling the execution of code for each index within the array. Let's harness its syntax:

const fruit = ['apple', 'orange', 'kiwi', 'banana'];

for (let i = 0; i < fruit.length; i++) {
    // code to execute for each index
}

Breaking it down:

  • Initialization: Begin by specifying where in the array you want to start. let i = 0 sets the initial index to zero (the first value).
  • Condition: Define where in the array you want to stop. Typically, we iterate up to the last value in the array, denoted by i < fruit.length.
  • Iteration: Specify how you want to traverse the array. Use i += 1 or the shorthand i++ to go through values one after another. Adjust as needed, like i += 2 for every other value or i += 3 for every third value, and so on.

For instance, if our goal is to print the value of each index in the fruit array using a for loop, the code would look like this:

for (let i = 0; i < fruit.length; i++) {
    console.log(fruit[i]);
}

JavaScript: Control Flow - Conditionals

September 9, 2023

In the vast landscape of programming, control flow serves as the director of the script, orchestrating the sequence of actions based on specific conditions or loops. At the heart of this control lies the powerful tool of conditionals, guiding our code through decision-making pathways.

if/else Statements

The cornerstone of conditionals is the if/else statement. Behold its syntax:

if (/* condition is true */) {
    // run this code
} else if (/* another condition is true */) {
    // run this code
} else if (/* yet another condition is true */) {
    // run this code
} else {
    // run this code as a catch-all
}

One of the beauties of if/else statements is their flexibility to handle various conditions. The code is read from top to bottom, and once a condition is true, the program executes the corresponding code and gracefully exits the set of statements.

Let's put this to the test with a day-of-the-week scenario:

// declare a variable as a test case
let day = 'Friday'

// create your statements
if(day === 'Monday') {
    console.log('Ugh. Back to work.')
} else if(day === 'Wednesday') {
    console.log('So close to the weekend!')
} else if (day === 'Friday') {
    console.log('It\'s the weekend, time for some self-care!!!')
} else {
    console.log('Just another day')
}

Remember, when comparing variables to values, use === to check for equality.

Additional Operators

Beyond equality, conditionals involve an arsenal of operators to perform comparisons:

  • > : greater than
  • < : less than
  • >= : greater than or equal to
  • <= : less than or equal to
  • !== : not equal to (! is known as the bang operator)

These are the basics of if/else statements and operators, laying the foundation for more advanced conditionals like ternary operators and switch statements. As you progress, these concepts will become your allies in crafting sophisticated JavaScript scripts.

JavaScript: Arrays

September 8, 2023

Arrays are the unsung heroes of programming. They're like versatile containers that help us store, organize, and manage data effortlessly. Let's dive into the world of JavaScript arrays and see how they work their magic.

Array Basics

In JavaScript, arrays are declared just like variables, but with a twist. They use square brackets to signify their array nature, like this:

let array = ['itemOne', 'itemTwo', 'itemThree'];

Each item in an array is stored sequentially and indexed, starting at 0 and ending at length - 1. Let's break down the array above:

  • 'itemOne' is at index 0
  • 'itemTwo' is at index 1
  • 'itemThree' is at index 2
  • While there are three elements in the array, the last index is always one less than the array's length.

Getting the Length

You can easily find out how many elements are in an array using the length property:

let array = ['itemOne', 'itemTwo', 'itemThree'];
array.length; // This will give you 3

Accessing Elements

To fetch a single element, use bracket notation. Specify the desired index inside square brackets. For instance, to access 'itemTwo' from our array:

array[1]; // This will give you 'itemTwo'

Array Superpowers

Arrays can do much more than storing data. They're your trusty sidekick for manipulating data. You can add, remove, or modify elements, among other things. Check out the resources below to explore these superpowers further.

Arrays might seem simple, but they're a fundamental building block in programming. Mastering them will open doors to more complex data manipulation and make your code more efficient. Happy coding! 🔥👩‍💻👨‍💻

JavaScript: Variables and Data Types

September 7, 2023

JavaScript empowers developers to bring interactivity to webpages through the execution of specific lines of code. However, working with the script.js file might not seem intuitive at first. When approaching coding, think of it as a written language rather than a spoken one. Most of the time, you must break down your actions into precise steps for the program to understand your intentions. For instance, while we casually say "one," in programming, we need to be explicit. We achieve this precision using variables.

What are Variables?

Variables are like containers that allow us to store and manage data in our programs. They are essential for making our code understandable and interactive.

  1. Declaration: We start by declaring a variable using keywords like "let" or "const" (we'll dive into these later).
  2. Descriptive Name: Next, we give our variable a meaningful name, something that reflects the data it holds. For example, let number.
  3. Equality: We use the equality symbol (=) to assign a value to our variable. For instance, let number = 1.
  4. Value: Finally, we specify the actual value the variable holds. In this case, we've told the program that there's a variable called "number," and it contains the value 1.

A quick note: You might come across the keyword "var" in older code, but it's considered outdated. Stick with "let" and "const" for modern JavaScript.

Data Types

Now that we know how to declare variables let's explore different types of data they can hold. Think of data types as the building blocks of your program. Here are some fundamental ones:

  • Number Data Type: Use this to represent numeric values. Numbers can be integers (whole numbers) or floating-point numbers (numbers with decimals). For example:
    let age = 25;
    let price = 9.99;
  • String Data Type: Use this to represent text, and enclose it in double quotes, single quotes, or backticks:
    let greeting = "Hello!";
  • Boolean Data Type: This type represents true or false values. It's often used for conditions and logic:
    let isCodingFun = true;
    let isRaining = false;
  • Null Data Type: Null represents the intentional absence of any object value. It's often used to signify that a variable should have no value:
    let emptyBox = null;
  • Undefined Data Type: Undefined is a variable that has been declared but hasn't been assigned a value yet. It's the default value of a declared variable:
    let notDefinedYet; // This is undefined

With these data types, we can build complex programs and make our code more expressive.


In JavaScript, you have two primary ways to declare variables: let and const. These keywords have different behaviors:

  • let: Variables declared with let can be reassigned. This means you can change their values after the initial assignment.
  • const: Variables declared with const are constant, meaning their values cannot be reassigned after the initial assignment. It's used for values that should remain unchanged throughout the program.

Let's illustrate this with examples:

let count = 10;
count = 20; // Valid, you can reassign 'count' with 'let'
console.log(count); // Outputs 20

const pi = 3.14159;
pi = 3.14; // Invalid, you can't reassign 'pi' with 'const'
console.log(pi); // Outputs an error

In the first example, count is declared with let, so you can change its value later. In the second example, pi is declared with const, so attempting to change its value results in an error.

Understanding when to use let and const is crucial for writing robust and maintainable code. Use let for variables that may change, and use const for values that should remain constant.

Practical Usage

Now that we have a bunch of variables, how do we visualize them? We use the console.log() function to display the value of a variable in the terminal. For instance, if we have a variable called greeting, we can see its value by typing console.log(greeting).

So, whether it's a number crunching or text processing, variables and data types are the foundation of JavaScript, allowing us to communicate precisely with our programs.

Planning - A Lesson in Patience

September 5, 2023

Software development is more than just coding. It's a collaborative effort that involves careful planning and coordination among different teams. In the world of technology, there are typically two approaches: one suggests meticulous planning to avoid potential problems like building the wrong features, excessive rework, complex solutions, and communication issues. The other approach believes in starting with an idea and allowing the project to evolve naturally. While both have their merits, most teams find that a well-thought-out plan ensures the success of their projects.

Product Management

Product ideas usually originate from the product management team. These ideas can involve creating a new product, adding features to an existing one, or refining existing aspects. Product managers act as advocates for customers within the organization, making sure that the market's voice is heard.

One effective method for the product team to turn their ideas into actionable plans is by creating user stories. User stories are informal explanations of a software feature, framed from the end user or customer's perspective. They aim to clarify how a piece of work will deliver specific value to the customer.

These stories use plain language without technical jargon to provide context for the development team. After reading a user story, the team understands why they are building something, what exactly they are building, and the value it will provide. User stories typically follow this format: "As a [persona], I [want to], [so that]."

For example, consider this user story: "As a frequent online shopper, I want to be able to save items to my wishlist so that I can easily find and purchase them without searching for them again."

Design

Once the product team has a clear idea, they collaborate with designers to visualize it. Designers play a pivotal role in ideation, definition, and early prototyping of the product. They take the initial idea and translate it into tangible web pages based on the product team's concepts.

Designers use tools like Figma to help create a visual representation of the final product's appearance and interactions.

Development Team

With a well-defined plan in hand, the development team takes over to build the product. This team includes developers responsible for the technical aspects, often alongside testing and quality assurance experts. They rely on the agreed-upon designs to guide their work, reducing guesswork.

Developers use tools such as Trello or Jira to break down projects into manageable tasks, often referred to as stories or tickets. When multiple developers collaborate on a project, it's efficient for each developer to focus on one part at a time. Tickets can be further divided into smaller tasks based on the team's preferences.

Project Management

Now, let's not forget the crucial role played by project management 😁. Project managers are the conductors of this intricate orchestra. They coordinate the various elements of multiple projects, ensuring that each team member is in sync. Their goal is to avoid downtime and missteps in the development process.

Project managers facilitate many of the important conversations throughout the project lifecycle. They act as liaisons between the product management, design, and development teams, ensuring that everyone is on the same page. They manage timelines, allocate resources, and identify potential bottlenecks before they become problems.


Tool Agnosticism: A Student's Advantage

As a software development student, your journey may start with solo projects where you explore various tools, libraries, and frameworks. However, as you enter the industry, you'll collaborate with diverse teams, each with its own set of preferences and tools. This is where being tool-agnostic becomes your secret advantage.

Being tool-agnostic means not overly relying on a single tool or technology stack. Instead, it's about having a flexible mindset that allows you to adapt to the tools used by your team or organization seamlessly. Here's why it matters:

  1. Smooth Collaboration: In a professional setting, software development is a collaborative effort. Your ability to adapt to different tools and workflows enables you to seamlessly integrate into a team. It ensures that you're not limited by your toolset when working with colleagues who might have different preferences.
  2. Versatility: The tech industry evolves rapidly. What's popular today might not be tomorrow. By being tool-agnostic, you become versatile and ready to embrace new technologies and methodologies as they emerge. This adaptability is a valuable trait in the ever-changing world of software development.
  3. Problem-Solving: Different projects come with different requirements. Some tools are better suited for specific tasks than others. Being tool-agnostic empowers you to choose the right tool for the job, enhancing your problem-solving skills and efficiency.

So, as a student, embrace the freedom to explore various tools and technologies. Build a strong foundation in the fundamentals of programming and development principles. But also, keep in mind that your journey doesn't end there. The ability to adapt and collaborate effectively with diverse teams will give you a significant advantage when you step into the industry. It's not just about what you know; it's about how well you can have meaningful conversations and contribute to the collective success of your team.


There isn't one prescribed way for these teams to work together. Each team finds an approach that suits them best, but it's essential to understand why planning at every phase is crucial. At the end of the day, it's a collaborative effort, and the more we can communicate and document our progress, the better off the team will be. The delicate balance between structured planning and agile project management is a conversation for another day.

Getting Started with Git and GitHub for Web Development

September 4, 2023

You've just created a stunning webpage, and now it's time to share it with the world. Enter Git and GitHub – your allies in the journey of version control and collaborative web development.

Git is a superhero among version control systems. It helps you keep track of changes to your files over time, a lifesaver when you're working with others or need to revert to a previous version of your code. GitHub, on the other hand, is the ultimate platform built around Git, offering online storage, collaboration tools, and version tracking.

Git and GitHub are not the same!

Before diving in, let's get familiar with some crucial terms:

  • Git Repository: Think of it as a time machine for your project, storing every change, who made it, and when. It's your ticket to accessing past versions of your code.
  • Git Branches: These allow multiple developers to work on the same codebase without chaos. The "main" branch holds the stable code.
  • Local Repository: Your playground for coding, where you interact with Git on your machine.
  • Remotes: These are Git repositories typically hosted online (like on GitHub). They keep your local repository in sync with the remote one.
  • Forking: Copying a repository to your own GitHub account. It's handy for making changes without affecting the original.
  • Cloning: Making a local copy of a remote repository. It's like having a duplicate of someone's project on your machine.
  • Pushing: Sending your local changes to the remote repository, akin to launching an aircraft on a journey.
  • Pulling: Fetching and merging changes from the remote repository into your local one, ensuring your code is up-to-date.

Setting up Git and GitHub is quite involved, so you’ll need to following the documentation to get those set up!

When starting a new project, we want to turn a folder/directory into a Git repository, allowing you to track changes within it. Navigate to the project folder in your terminal and run git init.

At the same time, let's create a new repository on GitHub.

  1. Log in to your GitHub account.
  2. Click on the "+" sign in the top-right corner of the page and select "New repository."
  3. Choose a name for your repository, such as "personal-website."
  4. Optionally, you can add a description for your repository.
  5. Keep the repository as public.
  6. Click on the "Create repository" button.

Now that we’ve made changes to our project locally we want to update our GitHub repository to reflect those changes. We now have to go through a series of commands in our terminal. This process is analogous of going from one location to another via an airplane.

  • After making changes locally, use git add . to stage them.
    • Git add is like getting all the passengers scanned and boarded on the aircraft. The aircraft is still at the gate.
  • The next command is git commit -m "<insert a descriptive message>"
    • The aircraft has left the gate, and is on the runway. Waiting to take-off.
  • To push your changes to GitHub, run git push <remote-name> <remote-branch-name>. git push origin main (for starters).
    • Imagine your aircraft taking off, in transit, landing, taxiing, and deboarding at its destination.

With your GitHub repository now mirroring your local one, remember to commit changes consistently and push them to your remote repository. Git and GitHub are powerful tools, and mastering them will elevate your web development game.

Tips & Tricks

  • git init - Start a new Git repository for your project.
  • git add . - Stage changes for the next commit.
  • git commit -m "commit message" - Create a checkpoint with a descriptive message.
  • git push <remote-name> <remote-branch-name> - Send local commits to a remote repository.
  • git clone <repository-url> - Make a local copy of a remote repository.
  • git pull <remote-name> <remote-branch-name> - Fetch and merge updates from a remote repository.

CSS: The Magic Wand of Web Styling

September 3, 2023

Have you ever wondered how websites achieve their stunning visual designs? The secret sauce is Cascading Style Sheets (CSS), a powerful language that brings life and style to the structure provided by HTML. It's what allows you to control the layout, colors, fonts, and other visual aspects of your web pages. In this journey through the world of CSS, we'll explore how it can transform a plain webpage into a visually captivating masterpiece.

There are a few ways to add CSS styles to your HTML documents, but today, we'll focus on the external stylesheet method.

To get started, you need to create an external stylesheet. Here's how:

  1. Navigate to Your HTML Folder: Find the folder where your HTML file is located.
  2. Create a New File: In that folder, create a new file and name it styles.css. This file will be your styling headquarters, where you'll work your magic.

Now, here's where the magic happens: we need to link this stylesheet to our HTML document. We do this using the <link> tag.

<link rel="stylesheet" href="styles.css">

The new element defines a relationship and the location of the relationship relative to the HTML file location. This new element should be placed between the <head> tags and above the <title> tag.

<head> <meta charset="UTF-8"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="styles.css"> <title>CSS: The Magic Wand of Web Styling</title> </head>

Now that your HTML and CSS are connected, it's time to sprinkle some styling magic. We use CSS selectors to target HTML elements and apply styles. For example, you can change the background color of your <header> like this:

header { background-color: gray; }

Let's enhance our webpage even further. How about changing the background color of the footer to black and the text to white? Here's how you can do it:

header { background-color: gray; } footer { background-color: black; color: white; }

With CSS, you've got the magic wand to create web designs that dazzle and inspire. So go ahead, unleash your creativity and make your web pages truly shine!

HTML: The Art of Structuring Your Webpage

September 2, 2023

Have you ever looked at a webpage and wondered how it's all organized? The secret lies in the structure, and in the web development realm, that structure is built with HTML. Today, we'll embark on a journey to create a basic webpage structure and then refine it using semantic HTML tags.

In the vast landscape of web development, starting with a solid structure is crucial. The way you organize your content not only affects how your webpage looks but also how it performs in search engines and how accessible it is to everyone. Let's dive in!

We'll begin with a simple webpage that is structured using <div> elements. These elements are like building blocks that help us organize content. Here's what our initial HTML structure will look like:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>The Art of Structuring</title> </head> <body> <div> <h1>Welcome to My Webpage</h1> </div> <div> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> <li><a href="#">Contact</a></li> </ul> </div> <div> <h2>About Me</h2> <p>This is where you'll learn all about me.</p> </div> <div> <p>© 2023 My Webpage</p> </div> </body> </html>

To make our structure even more understandable, we can add unique identifiers using the id attribute to each <div> element. This helps with code readability and makes it easier to target specific parts of our webpage programmatically.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>The Art of Structuring</title> </head> <body> <div id="header"> <h1>Welcome to My Webpage</h1> </div> <div id="nav"> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> <li><a href="#">Contact</a></li> </ul> </div> <div id="main"> <h2>About Me</h2> <p>This is where you'll learn all about me.</p> </div> <div id="footer"> <p>© 2023 My Webpage</p> </div> </body> </html>

In a well-structured webpage, the header typically features a warm welcome or branding at the very top. The navigation section, indicated by the <nav> element, defines links that help users navigate your site. The main content area, enclosed within the <main> element, holds the bulk of your webpage's content. Lastly, the footer, positioned at the bottom, often contains supplementary information and links.

While <div> elements provide structure, HTML5 introduced a set of semantic tags that add meaning to your webpage's structure. Let's refactor our initial HTML using these semantic tags for a more accessible and SEO-friendly result.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>The Art of Structuring</title> </head> <body> <header> <h1>Welcome to My Webpage</h1> </header> <nav> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> <li><a href="#">Contact</a></li> </ul> </nav> <main> <h2>About Me</h2> <p>This is where you'll learn all about me.</p> </main> <footer> <p>© 2023 My Webpage</p> </footer> </body> </html>

By using semantic tags like <header>, <nav>, <main>, and <footer>, we've not only improved the clarity of our code but also made it more accessible to users and search engines. Semantic HTML tells browsers and screen readers what each section of your webpage represents.

Tips & Tricks

  • Understanding how to structure your content effectively is a fundamental skill in web development.
  • Start Big, Then Refine: Consider structuring your webpage by starting with larger containers (header, main, footer) and then breaking them down further:
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Document</title> </head> <body> <header></header> <main></main> <footer></footer> </body> </html>

HTML Basics: Getting Started with Web Development

September 1, 2023

Ever wondered how words and images seamlessly materialize on our browsers? If you're curious about the inner workings of the web, you're in for a treat! Let's embark on a journey to explore the foundational language that brings web content to life: Hyper Text Markup Language (HTML).

When you navigate the web or interact with websites, you're interacting with HTML. It's like the blueprint of a webpage, defining the structure and layout of the user experience on the other side of the screen.

To kickstart your HTML coding journey, you'll need a trusty code editor. My personal favorite? Visual Studio Code (VS Code). While you wait for the download, why not create a folder named 'Intro to HTML' on your 'Desktop'? This is where you'll practice your newfound skills using the Terminal. We'll come back to that folder shortly. Once VS Code is downloaded, open it up and explore the interface. Remember, always open the folder you're working with by navigating to File → Open Folder…. Locate your 'Intro to HTML' folder and dive in. You're now primed to craft your very first .html file!

The VS Code Explorer on the left side will showcase the folder you've just opened. Click on the folder, revealing several options. Opt for New File, and name it index.html. Voilà! You've just authored an .html document. Now, let's conjure up some magic.

Type in Hello World into the index.html file. Now, journey to your desktop, crack open the 'Intro to HTML' folder, and double-click the index.html file. Watch as a new browser window (I prefer Chrome) springs to life, displaying the illustrious words Hello World. Congratulations, magician!

But wait, it's not all enchantment and ease...

The index.html file must be structured in a way that browsers can decode its contents. It craves the following elements:

  1. <!DOCTYPE html> → A nod to the browser about the document type ahead.
  2. <html></html> → The grand container of all HTML elements.
<!DOCTYPE html> <html></html>

Imagine <html></html> as a treasure chest cradling all the secrets necessary to whisk the contents of index.html into a visually captivating web page. But, we're not done yet.

  1. <head></head> → A chamber for metadata.
  2. <body></body> → The sanctuary for all our content.
<!DOCTYPE html> <html> <head></head> <body></body> </html>

Within the head tags resides metadata that defines the document you're conjuring. This metadata won't be visible on the webpage.

The body tags, on the other hand, house the content destined for display in the browser.

As we sculpt the document's architecture, we add layers of detail, allowing the browser to interpret and enchant. I'll adorn the head with some metadata, but I encourage you to dive deeper into each element's significance.

<!DOCTYPE html> <html> <head> <meta charset="UTF-8"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Intro to HTML</title> </head> <body></body> </html>

Now we are ready to add content!

Let's summon the following between the body tags: <h1>Hello World</h1>.

HTML wields an arsenal of tags to conjure diverse elements. These tags nestle within angle brackets (<>). Behold a few often-used spells:

  • <h1> to <h6>: Headings of varying importance, with <h1> as the mightiest and <h6> as the meekest.
  • <p>: A paragraph of text.
  • <img>: Summons an image to grace your web page.
  • <a>: Weaves a hyperlink to another realm.

MacOS Terminal

August 31, 2023

Picture a green font against a black backdrop – a scene often portrayed in movies involving hackers. These visuals are always captivating, yet they might seem distant for the average person. Join me as we take a step closer to realizing your 'hacker-esque' dreams.

The terminal is a tool developers use to execute instructions on a computer. It provides text-based access to the operating system. Renowned for its speed, the terminal offers a plethora of utilities, from navigating files and folders to running scripts that automate tasks. Let's embark on our first journey into the world of the MacOS terminal.

To open the terminal, simply use Spotlight Search (press command + spacebar), type 'terminal,' and hit return. Voila! A new terminal window should materialize on your screen. For many of us, the terminal might lack the style of the green/black design mentioned earlier; instead, it could be a white screen with black font. You can always go to Terminal → Settings, click on the Profile tab, and personalize the background and font color to your liking.

Now that your terminal has a touch of swag, let's attempt to grasp how it functions. When I contemplate writing code or how the computer interprets a user's actions, I visualize every click I've made from the moment the computer is turned on to my current state. For instance, turning my computer on and logging into my account means I've accessed my MacOS and then my user account. So, if you were to type pwd into your terminal and press return, an address will be displayed – usually something like /Users/<your-username>. The command pwd shows your current location in the terminal; it stands for "print working directory." Let's break down our location or path: /Users/<your-username>.

  • The / symbolizes the root directory. All paths start from the root directory, analogous to booting your computer.
  • /Users signifies that you've entered the Users directory – the screen displaying all users on the computer.
  • /Users/<your-username> implies that you've logged into a specific account. This becomes your starting directory or path every time you launch your terminal.

Before we proceed, let's pause for a quick sidebar. There are a few terms we should acquaint ourselves with:

  • Directory: Often used interchangeably with 'folder.' While not precisely the same, for now, let's assume they are: directory = folder.
  • Path: The location of a file or folder in a file system (e.g., /Users/<your-username>). I begin in the root directory (/), move into the Users directory (/Users), and then arrive at a user directory (/User/<your-username>).
  • Absolute Path: The exact location of a file or folder from the root directory (e.g., /User/<your-username>).
  • Relative Path: A reference to a file or folder relative to your current position.

Feeling a bit perplexed? Don't worry; it'll make sense soon. Let's return to the main session.

Now that we've logged into our account and used the pwd command to confirm our location, let's forge ahead. I can now type ls into my terminal to view the contents of the current directory. If I want to enter or access the Desktop directory, I can either:

  1. Use the absolute path: cd /Users/<your-username>/Desktop
  2. Use the relative path: cd Desktop

The text to the left of the cursor indicates your current directory. You can always use pwd to see the absolute path.

Let's take a quick peek into how the temrinal compares to our computer's Graphical User Interface (GUI):

  • Type pwd into the terminal, to view your location
  • Type ls into the terminal, to list the contents in the current directory
  • Type cd Desktop to enter the 'Desktop' directory
  • Create three folders in the 'Desktop' directory --- Type mkdir folderName --- Keep an eye on the GUI!
    • mkdir FolderOne
    • mkdir "Folder Two"
    • mkdir "Folder\ Three"
  • Type ls into the terminal, to list the contents in the current directory
  • Type cd FolderOne to enter the 'FolderOne' directory
  • Type touch document.txt, to create a new file with the .txt extension --- Double click the respective folder in your GUI to see the results
  • Repeat the above step to create a file with a .pdf extension
  • Type cd .. into the terminal to exit the current directory and return to the 'Desktop' directory
  • You try! --- add a file to another folder.

Congratulations, you're now a few steps closer to becoming a programming wizard. Keep exploring your terminal and experiment with the commands in the next section.

Tips & Tricks

Command Explanation
pwd Shows the absolute path of your current directory
ls Displays files and folders in the current directory
cd Changes directories
cd .. Moves you back one directory
cd ~ Takes you to your home directory
mkdir Creates a new folder
touch Generates an empty file
cp Copies a specified file and pastes it with a new name
mv old/path new/path Moves a specified file to a designated location
mv path/filename path/newname If paths are the same but with a different file name, the file's name changes
rm Deletes the specified file
history Displays the history of previous commands
man Opens a help manual for a particular command
